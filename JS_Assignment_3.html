In JavaScript, they both are related to memory space and there is a fine line difference between them.

Difference between Undefined and Not defined variables in JS is as follows :-

NOT DEFINED

UNDEFINED

If the variable name which is being accessed doesn’t exist in memory space (not yet declared), then it would be not defined.

If it exists in memory space but hasn’t been assigned any value till now, then it would be undefined.

In JavaScript, a reference error is thrown when someone tries to accesses the variable which is not inside memory heap.

It is a JavaScript keyword that has a similar meaning as NULL. Everything which gets a space in memory will be undefined until we assign a value to that memory space.
